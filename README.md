# AI Dubbing — Chatterbox batch dubbing pipeline

Batch-replace the original foreign-language audio in a folder of videos with an
English voiceover generated by [Chatterbox TTS](https://github.com/resemble-ai/chatterbox)
(Resemble AI, MIT), using each video's matching `.srt` for text and timing.
Output is a folder of dubbed **MP4s whose length matches the originals**, with a
single **AAC** default audio track that plays everywhere.

> **Status: v1.0 (MVP).** Full audio replacement — clean English over silence.
> Preserving original background music/SFX (source separation) is intentionally
> deferred to a later release; see [Background music](#background-music--sfx-the-one-real-decision).

---

## How it works (per video)

1. **Parse** the `.srt` into ordered cues `(index, start, end, text)`.
2. **Generate** a Chatterbox TTS clip per cue (24 kHz mono), resampled to a
   common 48 kHz.
3. **Fit each clip to its time window.** Allotted = gap to the next cue (or the
   cue's own span). Too long → speed up with ffmpeg `atempo`, **capped at 1.5×**
   (`max_atempo`) to stay natural; too short → *slowed* to fill its slot
   (`min_atempo`, default `0.8`) so narration doesn't leave gaps between lines,
   with the leftover (real pauses) padded by the silent canvas. Every cue that
   exceeds the speed-up cap is logged for review.
4. **Place clips by absolute timecode** onto a silent canvas exactly the length
   of the source video — *no concatenation*, so the dub can't drift out of the
   runtime no matter how each cue lands.
5. **Loudness-normalize** the assembled track (`loudnorm`, −16 LUFS).
6. **Mux**: copy the original video stream untouched, attach the new English
   track as **AAC**, drop the original audio, mark the new track **default**.

The "place by absolute timecode on a full-length canvas" step is what keeps the
dub in sync; it's done in NumPy (sample offset = `round(start × 48000)`), which
is simpler and more robust than a hundreds-of-inputs `adelay`+`amix` filtergraph.

## Install

Requires **Python 3.10+**, **ffmpeg** on `PATH`, and (recommended) an NVIDIA GPU.

```powershell
# Windows / PowerShell — creates .venv and installs CUDA torch + everything else
powershell -ExecutionPolicy Bypass -File .\setup.ps1
# CPU-only:
powershell -ExecutionPolicy Bypass -File .\setup.ps1 -Cpu
```

Manual install:

```bash
python -m venv .venv && . .venv/Scripts/activate      # (Windows)
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu124
pip install -r requirements.txt
```

The first real run downloads the Chatterbox weights (~1–2 GB) from Hugging Face.

## Usage

```powershell
# 0) See how videos pair with subtitles — generates nothing:
python dub.py --dry-run

# 1) Single-video sanity check (recommended first), fast preview of first 15 cues:
python dub.py --only 01 --max-cues 15

# 1b) Full single video, with a cloned narrator voice held constant everywhere:
python dub.py --only 01 --reference-wav voice.wav

# 2) Full batch — idempotent: re-running skips videos already in output/:
python dub.py

# Other handy flags:
python dub.py --device cpu                 # no GPU
python dub.py --overwrite                  # re-dub everything
python dub.py --config config.yaml         # load knobs from YAML
```

### Faster: parallel GPU workers

A single Chatterbox stream only drives an RTX 4080 to ~35% (generation is
autoregressive, one cue at a time), so the batch is GPU-idle-bound, not
memory-bound. `run_parallel.py` runs several workers over disjoint videos, each
capped to a VRAM fraction so they co-fit — ~2× throughput at 2 workers (~80% GPU,
~13 GB/16 GB):

```powershell
python run_parallel.py --config config.yaml --workers 2
```

Idempotent like the normal batch (skips finished videos, resumes from cache).
Each worker logs to `logs/parallel_worker<i>.log`. On 16 GB, 2 workers is the
sweet spot; 3 tends to saturate compute anyway and risks OOM.

Run `python dub.py --help` for the full flag list.

### The dataset this ships configured for

The `mogoon/` course pairs videos to subtitles **by episode number**, not stem:

```
mogoon/01 Understanding Commercial Art.m4v   <->   mogoon/EN SUBS/[EN]Mogoon_01.srt
mogoon/Bonus 1 In-Depth Coloring Insights.m4v <-> mogoon/EN SUBS/[EN]Mogoon_BONUS01.srt
```

`match_strategy: episode` (the default) handles this. For folders that use the
classic `lecture01.mp4 ↔ lecture01.srt` convention, pass `--match stem`.

## Configuration

All knobs live in [`dubbing/config.py`](dubbing/config.py) and are documented in
[`config.example.yaml`](config.example.yaml). Resolution order:
**defaults → YAML (`--config`) → CLI flags**. Highlights:

| Knob | Default | Meaning |
|---|---|---|
| `reference_wav` | `null` | 5–10 s clip cloned as the narrator across all videos |
| `translate` | `false` | run each cue through a swappable translator first (see below) |
| `exaggeration` / `cfg_weight` | `0.5` / `0.5` | Chatterbox expressiveness / pacing |
| `max_atempo` | `1.5` | hardest speed-up before a cue is flagged |
| `min_atempo` | `0.8` | gap-fill: slow short clips to fill their slot (1.0 = off; lower fills more) |
| `extend_to_next_cue` | `true` | use the gap to the next cue as each window |
| `sample_rate` / `channels` | `48000` / `2` | working + output audio format |
| `device` | `cuda` | `cuda`/`cpu` (auto-falls back to CPU) |
| `max_cues` | `null` | preview: voice only the first N cues per video |

### Translation (off by default)

The SRTs are assumed already English, so the default path does **not** translate.
Set `translate: true` and replace the body of `translate_text()` in
[`dubbing/translate.py`](dubbing/translate.py) with your LLM/API call — the
signature is stable so nothing else changes. (For dubbing into a *non-English*
language you'd swap the base model for `ChatterboxMultilingualTTS`; the base
English model used here is correct for the translate-**to**-English path.)

## Output, logs & resuming

- **Output:** `output/<original name>.mp4` — video copied losslessly, new AAC
  voiceover as the single default track.
- **Resumable two ways:** finished videos are skipped (unless `--overwrite`),
  and every generated cue is cached under `.dub_cache/`, so a crash mid-video
  only regenerates the missing cues.
- **Per-run report** in `logs/`:
  - `run_<ts>.log` — full trace,
  - `run_<ts>_summary.json` — per-video status/counts,
  - `run_<ts>_capped.csv` — every cue that exceeded `max_atempo` (review these
    for squeezed speech).

> Chatterbox stamps each clip with Resemble's inaudible **Perth watermark** by
> design. It's preserved through atempo/loudnorm/mux and doesn't affect playback.

## Background music & SFX (the one real decision)

This pipeline **fully replaces** the original audio, so anything that wasn't
speech — music, ambience, SFX — disappears with the foreign voice. For
talking-head / lecture / interview content (like this course) that's ideal:
clean English over silence.

If you have videos with music/atmosphere worth keeping, full replacement will
feel bare. Preserving it needs a vocal/instrumental **source-separation** step
(e.g. [Demucs](https://github.com/adefossez/demucs)) to strip just the original
voice and keep the backing track, then mix the English over that. That's planned
for a later release (1.1/1.2), not v1.0.
